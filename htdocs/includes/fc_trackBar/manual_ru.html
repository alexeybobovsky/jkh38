FC-TrackBar manual

О проекте

FC-TrackBar (FastCoder Trackbar) — небольшая библиотека, позволяющая разместить и настроить на страницах ваших сайтов слайдеры. Единственное необходимое условие для нормальной работы трекбара - включенный в браузере JavaScript.

Установка

Скачайте архив с последней стабильной версией дистрибутива и распакуйте его в нужной директории. Подключите файлы trackbar.css и trackbar.js:

<link rel="stylesheet" type="text/css" href="trackbar.css" />
<script type="text/javascript" src="trackbar.js"></script>
Также необходимо наличие папки [imgtrackbar], которая содержит несколько изображений для слайдеров.

В архив включены демонстрационные файлы bind.html и example.html и папка [jQuery] с плагином для одноименного фреймворка.

Трекбар можно установить на страницу 2-мя способами:

  1. Задать HTML-элементу, в котором планируется размещение слайдера идентификатор (id) и выполнить в любом месте странице следующий сценарий:
    trackbar.getObject('one').init(
         {
              /* Параметры инициализации и функция обратного вызова */
         },
         "elementId"
    );
    Метод trackbar.getObject() принимает в качестве единственного аргумента уникальное имя для трекбара (это необходимо, когда на одной странице несколько слайдеров). Далее метод init() принимает в качестве первого аргумента объект в JSON-формате с найтройками, которые подробно описаны ниже, и id HTML-элемента.
  2. При втором способе установки не нужно указывать идентификатор HTML-элемента. Скрипт инициализации ставится прямо в том месте кода, где планируется появление трекбара.
    <script type="text/javascript">
    trackbar.getObject('one').init(
         {
              /* Параметры инициализации и функция обратного вызова */
         }
    );
    </script>

Настройка

Все параметры для настройки указываются при инициализации трекбара. Например:
trackbar.getObject('one').init(
     {
          width : 250, // px
          leftLimit : 0, // unit of value
          leftValue : 20, // unit of value
          rightLimit : 100, // unit of value
          rightValue : 80 // unit of value
     },
     "elementId"
);

Параметры задаются в формате [имя] [двоеточие] [значение] и перечисляются через запятую. После последнего запятую ставить не нужно.

Можно не задавать ни единого параметра, т.к. все свойства имеют значения по-умолчанию. В этом случае будет виден слайдер шириной 100пикселей с интервалом значений от 0 до 100.

В параметрах можно назначать не только свойства, но и методы, для динамичного управления трекбарами и сбора полученных значений.

API

Свойства

  • width — целочисленное свойство, задает длину трекбара в пикселях. По умолчанию — 100.
  • leftLimit — минимальное граничное значение для левого бегунка в абсолютных значениях. По умолчанию — 0.
  • leftValue — выставляемое значение для левого бегунка. Не может быть меньше минимального (leftLimit). По умолчанию — 0.
  • rightLimit — максимальное граничное значение для правого бегунка в абсолютных значениях. По умолчанию — 100.
  • rightValue — выставляемое значение для левого бегунка. Не может быть больше максимального (rightLimit). По умолчанию — 100.
  • dual — при выставлении в false два бегунка объединятся в один и будут бегать с единым значением. Выставление дефолтных параметров для объединенного бегунка происходит по значениям для левой части. Параметр rightValue просто игнорируется, даже если задан. По-умолчанию всегда выставлен true.
  • roundUp — параметр для округления (целое положительное число). Для того, чтобы ваши значения пробегали с определенным интервалом - укажите этот интервал. Например, если задать roundUp : 50, то значения будут 0, 50, 100, 150... По-умолчанию не задан.
  • clearLimits — при выставлении true не будут выводиться серые крайние значения. По-умолчанию всегда false.
  • clearValues — при выставлении true не будут выводиться значения над бегунками. По-умолчанию всегда false.

Методы

  • onMove — принимает в качестве значения функцию, которая через this имеет доступ ко всем свойствам объекта трекбара и может в процессе движения работать со значениями this.leftValue и this.rightValue. Так, в примере есть серия из трех бегунков, которые изменяют при движении RGB-компоненты цвета фона и ДИВа.
    Также возможен вызов двух методов в контексте объекта:
    1. this.updateLeftValue — принимает в качестве единственного аргумента значение для левого бегунка и изменяет состояние трекбара в соот-ии с ним. Используется для синхронизации трекбаров (см bind.html).
    2. this.updateRightValue — mdash; принимает в качестве единственного аргумента значение для правого бегунка и изменяет состояние трекбара в соот-ии с ним. Используется для синхронизации трекбаров (см bind.html).
    По умолчанию пустая анонимная функция.
